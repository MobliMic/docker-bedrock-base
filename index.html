Docker and Bedrock

The purpose

  • For development use
  • May be for production

Why use bedrock?

  • Dependancy management
  • Makes version control easier
  • Better security
  • Better project structure
  • dotenv configuration

My container structure

  • Database
  • Application

services:
db:
image: mariadb
volumes:
 - db_data:/var/lib/mysql
restart: always
ports:
 - "33063:3306"
environment:
 MYSQL_ROOT_PASSWORD: root
 MYSQL_DATABASE: wordpress
 MYSQL_USER: wordpress
 MYSQL_PASSWORD: wordpress
		
project:
	depends_on:
		- db
	build:
		context: ./project
	volumes:
		- ./project:/var/www/html
	ports:
		- "8080:80"
		- "4434:443"
	restart: always
	links:
		- db
		
volumes:
	db_data:
		
mysqldump -h 127.0.0.1 -P 33063 -u root -p wordpress > dump.sql
